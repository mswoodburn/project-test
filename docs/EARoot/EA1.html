<b>CollectionObject </b>and <b>IdentifiableItem</b> can be used to represent different properties of what can be the same physical thing. <br/>All the links to the Collection Object are shown in the Overview, but many are missing from this diagram for clarity. Each relationship an entity that has with the Collection Object entities is described on those entities' diagrams.<br/><b>Concepts to Consider:</b><br/><ul>
<li><b>Object</b>: Anything considered and object will be a <b>CollectionObject</b> in this model.</li><li><b>Lot</b>: Any group of objects that will not be physically separated (10 fish in a jar) will be a single <b>CollectionObject</b>. Unsorted material could be a <b>CollectionObject</b>, linked to a single <b>IdentifiableItem</b>, or <b>CollectedMaterial</b>.</li><li><b>Acquisition lot</b>: The <b>Acquisition</b> will group these. <i>There may be a need for an AcquiredMaterial entity (though this might be pre-acquisition!)</i></li><li><b>Sample</b>: This will be another <b>CollectionObject</b>, with data in the <b>RelationshipType</b> on its creation.</li></ul>
<ul>
<li><b>Bulk sample</b>:  This will be <b>CollectedMaterial</b>.</li></ul>
<ul>
<li><b>Specimen</b>: Anything fitting the wide definition of specimen will probably be a <b>CollectionObject</b>.</li><li><b>Multi-specimen object</b>: This will be a single CollectionObject, with multiple linked <b>IdentifiableItem</b>s.</li><li><b>Multi-part object</b>: This will be linked <b>CollectionObject</b>s, with data in the <b>RelationshipType</b>s, linking to a single <b>IdentifiableItem</b>.</li><li><b>Material</b>: Covered by objects, lots, samples and bulk samples.</li><li><b>Derived object</b>: This is a <b>CollectionObject</b>, that is a child of another - the <b>RelationshipType</b> will define it as derived</li><li><b>Digital object</b>: This will be a CollectionObject, with a flag to indicate there is no physical object in the collection. The digital version might be in the CMS, or linked to (e.g. PID)</li><li><b>Determination</b>: This is show in the <b>Determination</b> entity.</li><li><b>Preparation</b>: This is a derived object, so a <b>CollectionObject</b>. details of how it was prepared from its parent will be in the <b>RelationshipType</b>.</li></ul><p/>
